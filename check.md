# CHECK — 시간표 블록 클릭 → 과목 카드 팝오버 (내가 이해한 것, 구현 전)

> 요청: 시간표에 추가된 과목 블록을 클릭하면, 그 과목을 **검색했을 때 나오는 카드**를 클릭한 자리에 띄운다. X/ESC로 닫는다. 카드의 **[추가하기] 자리에 [삭제]** 를 넣는다.

## 1. 기능 요약
- **내 시간표**의 과목 블록(`.tt-blk`)을 **클릭** → 그 과목의 **검색 결과 카드**(`render()`가 만드는 `.card`)를 **클릭한 위치에 팝오버**로 띄운다.
- 예) `물리전자 / 송장근` 을 추가 → 시간표의 물리전자 블록 클릭 → "물리전자" 검색 시 나오던 **송장근 교수님 카드**가 그 자리에 뜬다.
- 카드에는 검색 카드와 동일한 정보 표시: **과목명·학수번호·교수·학점·캠퍼스·시간 + 영역(`areaHtml`) 박스**.
- 카드 **닫기**: 카드 우측 상단 **X 버튼** + **ESC 키**. (추가로 카드 바깥 클릭 시에도 닫기 — 확인 필요)
- 카드 버튼: 검색 카드는 `[추가하기][담기][강의평]` 인데, 팝오버에서는 이미 시간표에 있는 과목이므로 **`추가하기` → `삭제`** 로 바꾼다. (담기·강의평은 그대로 둘지 확인 필요 — 일단 유지 가정)
- **삭제** 누르면: 그 과목을 현재 시간표에서 제거(`removeFromMyTable`) + 팝오버 닫기 + 시간표 다시 그림.

## 2. 코드 근거 (실제 확인함)
- 검색 카드 생성: `render(results)` → `content.js:368`. 카드 내부 = `.crow`(정보+`.actions`) + `areaHtml(c)`.
  - `.actions` 버튼: `추가하기`(data-add) / `담기`(data-bag) / `강의평`(data-review) — 지금은 결과배열 index로 배선.
- 시간표 블록 생성: `buildGrid()` → `content.js:665`. 각 블록 `b` = `{day,startMin,endMin,name,prof,color, key: keyOf(c)}` (`:657`). 블록 div `.tt-blk` 안에 `.rm`(× 제거) 버튼 존재(`:690`).
- **핵심**: 블록의 `b.key = keyOf(course)`. `myTable`엔 **추가할 때 전체 course 객체가 그대로 저장**(`myTable.push(course)` `:594`)됨. → **블록 key → myTable에서 원본 course 복원 가능** → 카드에 필요한 모든 필드(학점/캠퍼스/영역 등) 존재.
- 제거: `removeFromMyTable(key)` `:600` (해당 key 과목을 통째로 제거 → 그 과목의 모든 블록 사라짐).
- 영역 박스: `areaHtml(course)` `:318` — course만 주면 재생성 가능.

## 3. 구현 방향 (제안)
1. **카드 HTML 공용화**: 지금 `render()` 안에 있는 카드 마크업을 `courseCardHtml(course, mode)` 로 분리.
   - `mode='search'` → 버튼 `추가하기/추가됨` (index 배선)
   - `mode='block'` → 버튼 `삭제` (+ 담기/강의평) — 팝오버용
2. **블록 클릭 핸들러**: `renderTimetable()`에서 `.tt-blk` 에 click 리스너 추가.
   - `.rm`(×) 클릭은 `stopPropagation`(이미 있음) → 팝오버 안 뜸.
   - 블록의 `data-key`(신규로 넣거나 기존 `.rm[data-rm]`의 key 재사용) → `myTable`에서 course 찾기.
3. **팝오버 렌더**: shadow DOM에 떠있는 `.cardpop` 요소 하나 만들고, 클릭한 블록의 `getBoundingClientRect()` 기준 `position:fixed`로 그 근처에 배치(화면 밖으로 안 나가게 보정). 안에 `courseCardHtml(course,'block')` + 우상단 X.
4. **닫기**: X 클릭 / ESC keydown / (확인 시)바깥 클릭 → 팝오버 숨김. ESC는 기존 keydown 핸들러에 분기 추가(패널은 안 닫히게).
5. **삭제 배선**: 팝오버의 삭제 버튼 → `removeFromMyTable(key)` → 팝오버 닫기.

## 4. 확인하고 싶은 것 (구현 전)
- (a) 팝오버에 **담기·강의평 버튼도 유지**할지? (기본: 유지, `삭제`만 교체) — 아니면 삭제만 남길지.
- (b) **바깥 클릭으로도 닫기** 추가할지? (X + ESC 는 확정)
- (c) 삭제 시 **확인창(confirm) 없이 바로 삭제**할지? (블록 × 는 지금 바로 삭제됨 → 일관되게 바로 삭제 가정)

> 위 이해가 맞으면 (a)(b)(c) 답만 주면 바로 구현 시작. 기본 가정(담기·강의평 유지 / 바깥클릭도 닫기 / 확인 없이 삭제)으로 진행해도 되면 "그대로 진행"이라고만.
