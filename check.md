## A. 요청 4가지 (내가 이해한 것)
1. **강의평 버튼 외곽선 제거** — 지금 연분홍 배경 + 테두리(border)인데, **테두리 없애기**(배경만).
2. **"영역 정보 없음" 없애기** — 학사-전공/DS 과목이 현재 `영역 정보 없음`으로 뜨는 것 제거.
3. **학사-전공과목 → 영역 정보를 "주관학부-학과"로** 표시 (`skku-courses.json` 근거).
4. **메뉴 경로 + 세부구분 형태로 재구성** — "학사-OO과목"을 먼저 쓰고, 그 아래에 세부(교양 영역 / DS 기반·심화 / 전공 세부)를 쓰기.

## B. 데이터 근거 (실제 확인함)
- 런타임 데이터(`data/bundled-courses.json`)의 `isuType`(=ISU_NAME) 분포:
  `교양`, `기타`, `전공기반`, `전공심화`, `전공`, `전공(대학원)`, `실험실습`, `교직`, `DS기반(계열1/계열2/공통)`, `DS심화`.
- 교양만 INFORM에 `영역구분`이 있어 영역이 나옴. **전공·DS는 INFORM에 영역 없음 → 지금 "영역 정보 없음"으로 뜸.**
- **`skku-courses.json` = 학사-전공과목을 college→major로 묶은 데이터.** 과목마다:
  - 주관: `college.name`(예: 경영대학) + `major.name`(예: 경영학과) → **"경영대학-경영학과"**
  - 세부 분류 필드 3종: `isuType`(전공기반/전공심화/실험실습…), `areaGrp2`(전공핵심/전공일반…), **`areaGrp3`(전공코어/전공심화/실험실습)** ← 요청하신 "전공코어/전공심화/전공실험"과 가장 일치.
  - 조인 키 = `code`(학수번호). 분반 무관(같은 code면 동일 학과·세부).
- **DS·교직은 `skku-courses.json`에 없음**(그 파일은 전공만). DS 세부는 `isuType`("DS기반…"/"DS심화")에서 바로 뽑음.

## C. 표시 재설계 (제안)

카드의 영역 박스 = **위(작은 회색): 메뉴 경로 / 아래(큰 글씨): 세부**.

| 계열(isuType) | 위: 메뉴 경로 | 아래: 세부 표시 | 출처 |
|---|---|---|---|
| `교양`, `기타` | 학사-교양/기타과목 | 인문사회기반/자연과학기반, 인간/문화 등 (현행 유지) | INFORM 영역구분 |
| `전공기반`·`전공심화`·`실험실습`·`전공` | 학사-전공과목 | **주관학부-학과**(경영대학-경영학과) **＋ 세부**(전공코어/전공심화/실험실습) ← **둘 다** | `skku-courses.json` (code 조인, `areaGrp3`) |
| `전공(대학원)` | 학사-전공과목 | 주관학부-학과 ＋ 세부 자리에 `전공(대학원)` (areaGrp3 빈값) — **표시 확정** | `skku-courses.json` (조인됨) |
| `DS기반(계열1/2/공통)`·`DS심화` | 학사-DS과목 | **기반/심화 ＋ 계열** 둘 다 (예: "DS 기반 · 계열2") | `isuType` 파싱 |
| `교직` (75과목, `CFTD*`) | 학사-교양/기타과목(추정 ※E-6) | 세부=`교직` | 데이터 없음(주관학부-학과·영역 전무) |

**전공 예시(제안 레이아웃):**
```
학사-전공과목                 ← 위(작은 회색)
경영대학-경영학과              ← 아래(큰 글씨) = 주관학부-학과
전공코어                      ← 부가(작게) = 전공 세부구분(areaGrp3)
```
**DS 예시:**
```
학사-DS과목
DS 기반 · 계열2      (DS심화는 계열 없음 → "DS 심화")
```

## D. 구현 시 필요한 배선 (참고)

- 강의평 버튼: `.actions .review`의 `border` 제거(배경 `#fbeef1` 유지).
- `areaHtml()`에서 계열별 분기 + "영역 정보 없음" 분기 삭제.

## F. 구현 계획 (확정, E-6 답 후 착수)
1. `skku-courses.json` → `code → {college, major, areaGrp3, isuType}` **압축 룩업** 생성해 `data/`에 번들(전체 5MB 안 실음).
2. content.js `areaHtml()`를 계열별 분기로 재작성:
   - 교양/기타 → 현행(INFORM 영역)
   - 전공(대학원 포함) → 룩업 조인 → `학사-전공과목` + 주관학부-학과 + areaGrp3
   - DS → isuType 파싱 → `학사-DS과목` + 기반/심화 + 계열
   - 교직/그외 → 경로만(“영역 정보 없음” 문구 삭제)
3. CSS `.actions .review` 테두리 제거.
4. 검증: `node --check`, 파서 테스트, 브라우저 확인.
